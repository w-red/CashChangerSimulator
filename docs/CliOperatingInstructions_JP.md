# CLI 操作説明書 (Cli Operating Instructions)

CLI UI は、釣銭機シミュレーターをコマンドラインから操作・管理するためのインターフェースです。自動テストやスクリプトによる一括操作、および詳細なデバッグに最適化されています。

## 1. 起動方法

リポジトリのルートディレクトリから以下のコマンドで起動できます。

```powershell
dotnet run --project src/CashChangerSimulator.UI.Cli
```

## 2. 基本操作フロー

UPOS 標準のライフサイクル（Open → Claim → Enable）に従って操作します：
1. `open` : デバイスをオープン（初期化）します。
2. `claim` : デバイスの占有権を取得します。
3. `enable` : デバイスを使用可能状態（Enabled）にします。
4. （各種操作：入金、出金、在高確認など）
5. `disable` : デバイスを無効化します。
6. `release` : 占有権を解放します。
7. `close` : デバイスをクローズします。

## 3. コマンド一覧

起動後、以下のコマンドを入力して操作します。

### 基本・診断
| コマンド          | 説明                                                                 |
| :---------------- | :------------------------------------------------------------------- |
| `status`          | デバイスの状態（State）と現在の在庫（Inventory）の概要を表示します。 |
| `history [count]` | 最近の取引履歴を表示します（デフォルト 10 件）。                     |
| `help`            | 利用可能な全コマンドのヘルプを表示します。                           |
| `exit` / `quit`   | CLI アプリケーションを終了します。                                   |

### デバイス制御
| コマンド             | 説明                                          |
| :------------------- | :-------------------------------------------- |
| `open` / `close`     | デバイスのオープンとクローズ。                |
| `claim [timeout]`    | デバイスの占有権を取得（デフォルト 5000ms）。 |
| `release`            | 占有権の解放。                                |
| `enable` / `disable` | デバイスの有効化と無効化。                    |

### 現金操作
| コマンド                  | 説明                                                                 |
| :------------------------ | :------------------------------------------------------------------- |
| `read-counts`             | 現在の詳細な金種別在庫枚数を表示します。                             |
| `deposit [amount]`        | 入金を開始します。金額を指定するとランダムな金種が自動投入されます。 |
| `fix-deposit`             | 投入された現金を確定し、在高に反映させます。                         |
| `end-deposit`             | 入金取引を終了します。                                               |
| `dispense <amount>`       | 指定した金額を出金します。                                           |
| `adjust-counts <v:c,v:c>` | 在庫を強制的に上書きします（例：`1000:10,500:5`）。                  |

### 設定・システム
| コマンド             | 説明                                                    |
| :------------------- | :------------------------------------------------------ |
| `config list`        | 全設定項目と現在の値を一覧表示します。                  |
| `config set <k> <v>` | 設定値を一時的に変更します。                            |
| `config save`        | 変更した設定を `config.toml` に保存します。             |
| `log-level <level>`  | ログ詳細度（debug, info, warning, error）を変更します。 |
| `run-script <path>`  | 指定した JSON ファイルに従い、操作を自動実行します。    |

各コマンドの挙動や、初期在庫、しきい値などの詳細設定については、[設定ガイド](ConfigurationGuide_JP.md) を参照してください。

## 4. スクリプト実行 (run-script)

JSON形式のスクリプトファイルを使用して、複雑な操作を自動化できます。

### サンプルスクリプト (`sample.json`)

```json
[
  { "Op": "Open" },
  { "Op": "Claim", "Value": 1000 },
  { "Op": "Enable" },
  { "Op": "BeginDeposit" },
  { "Op": "TrackDeposit", "Currency": "JPY", "Type": "Bill", "Value": 1000, "Count": 1 },
  { "Op": "FixDeposit" },
  { "Op": "EndDeposit" },
  { "Op": "Dispense", "Value": 1000 }
]
```

## 5. ロギング
詳細なログは `logs/` フォルダに出力されます。`log-level` コマンドを使用して、実行時に出力レベルを変更可能です。

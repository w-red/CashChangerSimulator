# 設定ガイド (Configuration Guide)

本シミュレータは、`config.toml` ファイルを通じて高度なカスタマイズが可能です。WPF版アプリケーションでは、一部の設定をGUIから変更することもできます。

## 1. 設定ファイルの場所
実行バイナリと同じディレクトリにある `config.toml` が参照されます。ファイルが存在しない場合は、デフォルト設定で自動生成されます。

## 2. システム設定 (`[System]`)
アプリケーションの基本的な動作を規定します。

| キー           | 型     | デフォルト   | 内容                                   |
| :------------- | :----- | :----------- | :------------------------------------- |
| `CurrencyCode` | string | `"JPY"`      | 使用する通貨コード (`JPY`, `USD` 等)。 |
| `CultureCode`  | string | `"en-US"`    | 表示言語 (`ja-JP`, `en-US`)。          |
| `UIMode`       | string | `"Standard"` | UIの動作モード (`Standard`, `Pos`)。   |

## 3. シミュレーション設定 (`[Simulation]`)
デバイスの疑似的な挙動を調整します。

| キー              | 型   | デフォルト | 内容                                               |
| :---------------- | :--- | :--------- | :------------------------------------------------- |
| `DispenseDelayMs` | int  | `500`      | 払い出し操作完了までの疑似待ち時間（ミリ秒）。     |
| `HotStart`        | bool | `false`    | アプリ起動時に自動的に `Open` 状態にするかどうか。 |
| `CapRealTimeData` | bool | `true`     | リアルタイムデータ通知能力の有無。                 |

## 4. ロギング設定 (`[Logging]`)
診断ログの出力方法を制御します。

| キー            | 型     | デフォルト      | 内容                                                      |
| :-------------- | :----- | :-------------- | :-------------------------------------------------------- |
| `EnableConsole` | bool   | `true`          | コンソールへのログ出力。                                  |
| `EnableFile`    | bool   | `true`          | ファイルへのログ出力。                                    |
| `LogLevel`      | string | `"Information"` | 出力レベル (`Debug`, `Information`, `Warning`, `Error`)。 |
| `LogDirectory`  | string | `"logs"`        | ログの保存先ディレクトリ名。                              |
| `LogFileName`   | string | `"app.log"`     | ログファイル名。                                          |

## 5. 在庫としきい値の設定

### 共通しきい値 (`[Thresholds]`)
特定の金種で個別に設定されていない場合に使用されるデフォルト値。

| キー        | 型   | デフォルト | 内容                                     |
| :---------- | :--- | :--------- | :--------------------------------------- |
| `NearEmpty` | int  | `5`        | 在庫僅少（ニアエンプティ）とみなす枚数。 |
| `NearFull`  | int  | `90`       | 満杯間近（ニアフル）とみなす枚数。       |
| `Full`      | int  | `100`      | 満杯（フル）とみなす最大収容枚数。       |

### 金種別設定 (`[Inventory.<Currency>.Denominations.<ID>]`)
特定の金種（一万円札、百円玉など）に対する詳細な設定です。

| キー            | 型     | 内容                                                |
| :-------------- | :----- | :-------------------------------------------------- |
| `DisplayName`   | string | 英語の表示名。                                      |
| `DisplayNameJP` | string | 日本語の表示名。                                    |
| `InitialCount`  | int    | 起動時の初期枚数。                                  |
| `NearEmpty`     | int    | 個別のニアエンプティ判定値（`-1` で無効）。         |
| `NearFull`      | int    | 個別のニアフル判定値（`-1` で無効）。               |
| `Full`          | int    | 個別の最大収容枚数（`-1` で無効）。                 |
| `IsRecyclable`  | bool   | 払い出し（還流）に使用するか。 `false` で回収専用。 |
| `IsDepositable` | bool   | この金種の入金を許可するか。                        |

## 6. WPFアプリでの設定変更
WPF版では「System Settings」ウィンドウから、以下の操作が可能です。
1. **全般設定**: 通貨や言語、UIモードの変更。
2. **在庫調整**: 各金種の初期枚数やしきい値の編集。
3. **リセット**: 「Default」ボタンで工場出荷時の設定（デフォルトの `SimulatorConfiguration`）に戻すことができます。

> [!IMPORTANT]
> 通貨（CurrencyCode）の変更を反映させるには、アプリケーションの再起動が必要です。

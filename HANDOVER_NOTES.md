# プロジェクト引き継ぎ資料: CashChangerSimulator

## 1. プロジェクト概要
このプロジェクトは、OPOS規格に準拠した入出金機（Cash Changer）をソフトウェアでエミュレートするシミュレーターです。WPFによるUI、UPOSサービスオブジェクトの実装、およびテストスイートで構成されています。

## 2. 現在の状態
- **テスト合格率**: 100% (545/545 成功)
- **ビルドステータス**: すべてのプロジェクト（Core, Device, UI, Test）で正常にビルド可能。
- **UIオートメーション**: `FlaUI` を使用したUIテストが安定して動作。プロセス残留問題も解決済み。

## 3. 主要な修正・改善内容（直近の作業）
- **UIテストの安定化**:
    - `CashChangerTestApp` におけるOSレベルでのプロセスクリーンアップの実装。
    - ウィンドウ検出タイムアウトの最適化（30秒）。
- **非同期状態管理の改善**:
    - `DispenseController` において、`IsBusy` ステータスを実処理（await）の直前にセットし、テストでのレースコンディションを防止。
- **テストのカバレッジ向上**:
    - `IViewService` の導入により、ViewModel（Inventory, Dispense）の純粋なユニットテストを実現。
- **ビルド環境の整備**:
    - DocumentationTests, PerformanceTests のビルドエラーを修正し、全プロジェクトを通した検証を可能にしました。

## 4. 技術的な注意点と教訓
- **UIテスト実行時**:
    - テストランナー側で並列実行を無効にすることを推奨します（現在は `GlobalTestSettings.cs` で制御）。
- **プロセス管理**:
    - テストが不自然に失敗する場合、背景で `CashChangerSimulator.UI.Wpf.exe` または `testhost.exe` がゴーストプロセスとして残っていないか確認してください。
- **Claim（排他制御）**:
    - デバイスは `Claim` される必要があり、排他制御は `InternalSimulatorCashChanger` 層で担保されています。
